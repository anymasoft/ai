## ЧАСТЬ 8: ТОЧКА БЕЗУБЫТОЧНОСТИ (Break-Even)

### Какой процент пользователей может выбирать дорогие модели?

**Для Pro:**
```
Максимум можно потратить на себестоимость: $39.90
Чтобы остаться в маржин (70%): $39.90 × 0.3 = $11.97

Если 100% выбирают Claude Sonnet: себестоимость $51 > $11.97
Если X% выбирают Sonnet, остальные GPT mini:
  X × $51 + (100-X) × $2.10 ≤ $11.97
  51X + 210 - 2.1X ≤ 11.97
  48.9X ≤ -198.03
  X ≤ -4%

  РЕЗУЛЬТАТ: Невозможно обеспечить маржин 70% с Sonnet в Pro!
```

## ЧАСТЬ 9: РИСКИ И ПРОБЛЕМЫ

### 1. Выбор модели непредсказуем

**Проблема:**
- Пользователь может выбрать дорогую модель и превратить прибыль в убыток
- Нет контроля над выбором внутри тарифа
- При Sonnet маржа отрицательна

**Статус:** 🔴 КРИТИЧЕСКИЙ РИСК

### 2. Абьюз дешевых моделей

**Проблема:**
- Пользователь на Pro может делать 1000+ запросов в час
- Каждый запрос с GPT-4o mini = копейки себестоимости
- Но все равно прибыль, потому что маржа 94%

**Вывод:** Не является риском (вы все равно зарабатываете)

**Статус:** 🟡 ДОПУСТИМО

### 3. Абьюз дорогих моделей

**Проблема:**
- Пользователь может выбрать Claude Opus и делать 100+ запросов в день
- Каждый запрос = $0.034, лимит 5M токенов
- При 5M токенов себестоимость уже $85, а доход $39.90

**Вывод:** УБЫТОК гарантирован

**Статус:** 🔴 КРИТИЧЕСКИЙ РИСК

### 4. Контролировать нельзя

**Проблема:**
- Система биллинга уже отслеживает использование
- Но ограничить модели по тарифам нельзя (требует разработки)
- Пользователь свободно выбирает любую доступную модель

**Статус:** 🔴 АРХИТЕКТУРНОЕ ОГРАНИЧЕНИЕ
