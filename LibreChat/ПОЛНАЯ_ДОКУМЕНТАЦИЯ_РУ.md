## 🎯 Обзор проекта

### Что такое LibreChat?

**LibreChat** — это открытый, самостоятельно размещаемый AI чат платформа, которая объединяет все основные поставщики AI в единый, ориентированный на конфиденциальность интерфейс.

### Основные возможности:

- 🖥️ **UI в стиле ChatGPT** с улучшенным дизайном и функциями
- 🤖 **Мультимодельный выбор**: OpenAI, Anthropic (Claude), Google, Azure, AWS Bedrock, DeepSeek и др.
- 🔧 **Code Interpreter API**: безопасное, изолированное выполнение кода (Python, Node.js, Go, C/C++, Java, PHP, Rust, Fortran)
- 🔦 **Агенты и инструменты интеграции**: пользовательские помощники без кода, поддержка Model Context Protocol (MCP)
- 🔍 **Веб-поиск**: поиск в интернете для улучшения контекста AI
- 🪄 **Generative UI**: артефакты кода (React, HTML, Mermaid диаграммы)
- 🎨 **Генерация и редактирование изображений**: GPT-Image-1, DALL-E, Stable Diffusion, Flux
- 💾 **Пресеты и управление контекстом**: сохранение, поделка, форкирование сообщений
- 💬 **Мультимодальные взаимодействия**: загрузка изображений, чат с файлами
- 🌎 **Мультиязычный UI**: 40+ языков (включая русский)
- 🧠 **Reasoning UI**: динамический интерфейс для моделей с цепью размышлений (DeepSeek-R1)
- 🌊 **Resumable Streams**: восстановление потоков при разрыве соединения
- 🗣️ **Speech & Audio**: голос в текст и текст в голос (OpenAI, Azure, Elevenlabs)
- 📥 **Импорт/экспорт**: импорт из ChatGPT, экспорт как скриншоты, markdown, JSON
- 👥 **Мультипользовательская безопасность**: OAuth2, LDAP, Email login
- ⚙️ **Полная конфигурируемость**: Docker, Kubernetes, облачные развёртывания

## 🏗️ Архитектура

### Архитектура на уровне высокого уровня

```
┌─────────────────────────────────────────────────────────────────┐
│                     Frontend (React/TypeScript)                  │
│              /client и /packages/client                          │
│         (port 3090 dev server, port 3080 production)             │
└───────────────────────────────────────────────────────────────────┘
                               ↕ HTTP/WebSocket
┌─────────────────────────────────────────────────────────────────┐
│                 Backend (Node.js/Express)                        │
│              /api и /packages/api (TypeScript)                   │
│                    (port 3080)                                    │
└───────────────────────────────────────────────────────────────────┘
                               ↕
┌─────────────────────────────────────────────────────────────────┐
│            Shared Packages (TypeScript)                          │
│  ├─ /packages/data-provider (API типы и endpoints)              │
│  ├─ /packages/data-schemas (модели БД)                          │
│  ├─ /packages/api (переиспользуемые API компоненты)             │
│  └─ /packages/client (переиспользуемые UI компоненты)           │
└───────────────────────────────────────────────────────────────────┘
                               ↕
┌─────────────────────────────────────────────────────────────────┐
│            External Services & Databases                         │
│  ├─ MongoDB (основное хранилище)                                │
│  ├─ Redis (кеширование, сессии, resumable streams)              │
│  ├─ LLM Providers (OpenAI, Anthropic, Google и др.)             │
│  ├─ File Storage (local, S3, Firebase)                          │
│  └─ MCP Servers (для инструментов и расширений)                │
└───────────────────────────────────────────────────────────────────┘
```

### Монорепо структура

LibreChat использует **npm workspaces** для управления зависимостями между проектами:

```json
{
  "workspaces": [
    "api",           // Legacy Express server (JS)
    "client",        // React SPA
    "packages/*"     // Shared packages
  ]
}
```

**Принцип**: Весь новый код пишется на **TypeScript** в `/packages/api`, а `/api` остаётся минимальной JS оболочкой.

## 🚀 Установка и запуск

### Требования

- **Node.js**: v20.19.0+ или ^22.12.0 или >= 23.0.0
- **npm**: 11.10.0+
- **MongoDB**: для основного хранилища
- **Redis**: для кеширования и resumable streams (опционально)
- **Docker** (опционально): для контейнеризации

### 1. Клонирование репозитория

```bash
git clone https://github.com/danny-avila/LibreChat.git
cd LibreChat
```

### 2. Установка зависимостей

```bash
# Базовая установка
npm install

# Или умная переустановка (только если lockfile изменился)
npm run smart-reinstall

# Или полная переустановка (сотрёт node_modules)
npm run reinstall
```

### 3. Конфигурация

#### Скопируйте и отредактируйте .env файл:

```bash
cp .env.example .env
```

Минимальная конфигурация `.env`:

```env
# Сервер
HOST=localhost
PORT=3080
MONGO_URI=mongodb://127.0.0.1:27017/LibreChat
DOMAIN_CLIENT=http://localhost:3080
DOMAIN_SERVER=http://localhost:3080

# LLM провайдеры (выберите хотя бы один)
OPENAI_API_KEY=sk-...
ANTHROPIC_API_KEY=sk-ant-...

# Опционально: Redis
REDIS_URI=redis://127.0.0.1:6379
```

#### Конфигурируйте `librechat.yaml`:

```bash
cp librechat.example.yaml librechat.yaml
# Отредактируйте librechat.yaml для настройки моделей, интерфейса и т.д.
```

### 4. Запуск для разработки

#### Вариант 1: Раздельный запуск (рекомендуется)

```bash
# Терминал 1: Запустить backend (api сервер)
npm run backend:dev

# Терминал 2: Запустить frontend (dev server с hot reload)
npm run frontend:dev
```

- Backend: http://localhost:3080
- Frontend: http://localhost:3090

#### Вариант 2: С использованием Bun (быстрее)

```bash
# Терминал 1
npm run b:api:dev

# Терминал 2
npm run b:client:dev
```

### 5. Запуск для продакшена

```bash
# Собрать всё
npm run build

# Или собрать последовательно
npm run frontend

# Запустить backend в production
npm run backend

# Или используя Docker
docker-compose up -d
```

### 6. Проверка состояния

Откройте браузер:
- **Frontend**: http://localhost:3080 (в production)
- **Backend API**: http://localhost:3080/api/

## ⚙️ Конфигурация

### librechat.yaml - Основной файл конфигурации

Это YAML файл содержит всю конфигурацию приложения:

#### 1. **interface** — Настройки интерфейса

```yaml
interface:
  customWelcome: 'Welcome to LibreChat!'
  endpointsMenu: true
  modelSelect: true
  parameters: true
  sidePanel: true
  presets: true
  bookmarks: true
  multiConvo: true

  prompts:
    use: true
    create: true
    share: false
    public: false

  agents:
    use: true
    create: true
    share: false
    public: false

  fileCitations: true

  privacyPolicy:
    externalUrl: 'https://librechat.ai/privacy-policy'
    openNewTab: true

  termsOfService:
    externalUrl: 'https://librechat.ai/tos'
    openNewTab: true
    modalAcceptance: true
```

#### 2. **modelSpecs** — Доступные модели

```yaml
modelSpecs:
  enforce: true  # Только эти модели доступны
  list:
    # OpenAI
    - name: "gpt-4o"
      label: "GPT-4o"
      preset:
        endpoint: "openAI"
        model: "gpt-4o"
        temperature: 0.7
        max_tokens: 4096

    # Anthropic
    - name: "claude-opus-4-6"
      label: "Claude Opus 4.6"
      preset:
        endpoint: "anthropic"
        model: "claude-opus-4-6"
        temperature: 0.7
        max_tokens: 8192

    # DeepSeek
    - name: "deepseek-chat"
      label: "DeepSeek Chat"
      preset:
        endpoint: "deepseek"
        model: "deepseek-chat"
        temperature: 0.7
        max_tokens: 8192
```

#### 3. **balance** — Система баланса/кредитов

```yaml
balance:
  enabled: true
  startBalance: 15000        # Начальный баланс для новых пользователей
  autoRefillEnabled: false   # Автоматическое пополнение
  refillIntervalValue: 30
  refillIntervalUnit: 'days'
  refillAmount: 10000
```

#### 4. **endpoints** — Конфигурация AI провайдеров

```yaml
endpoints:
  # OpenAI
  openAI:
    apiKey: "${OPENAI_API_KEY}"
    baseURL: "https://api.openai.com/v1"
    models:
      default: ["gpt-4o", "gpt-4o-mini", "gpt-4-turbo"]
      fetch: true  # Загружать список моделей с API

  # Anthropic
  anthropic:
    apiKey: "${ANTHROPIC_API_KEY}"
    baseURL: "https://api.anthropic.com"
    models:
      default: ["claude-opus-4-6", "claude-sonnet-4-6", "claude-haiku-4-5"]
      fetch: true

  # Custom endpoint (совместимо с OpenAI)
  custom:
    - name: "deepseek"
      apiKey: "${DEEPSEEK_API_KEY}"
      baseURL: "https://api.deepseek.com/v1"
      models:
        default: ["deepseek-chat", "deepseek-reasoner"]
        fetch: false
      titleConvo: true
      titleModel: "deepseek-chat"
```

#### 5. **registration** — Регистрация пользователей

```yaml
registration:
  socialLogins: ['github', 'google', 'discord', 'openid', 'facebook', 'apple', 'saml']
  allowedDomains:  # Опциональный фильтр по доменам email
    - "company.com"
```

#### 6. **fileStrategy** — Хранилище файлов

```yaml
fileStrategy:
  avatar: "s3"        # Аватары пользователей → S3
  image: "firebase"   # Изображения в чате → Firebase
  document: "local"   # Документы → Local storage
```

## 📦 Рабочие пространства (Workspaces)

LibreChat использует монорепо с npm workspaces:

| Пространство | Язык | Сторона | Зависимости | Назначение |
|---|---|---|---|---|
| `/api` | JavaScript (legacy) | Backend | `packages/api`, `packages/data-schemas`, `packages/data-provider` | Express сервер (минимальные изменения) |
| `/packages/api` | **TypeScript** | Backend | `packages/data-schemas`, `packages/data-provider` | Новый backend код (только TS) |
| `/packages/data-schemas` | TypeScript | Backend | `packages/data-provider` | Модели БД, Mongoose схемы |
| `/packages/data-provider` | TypeScript | Shared | — | Типы API, endpoints, data-service |
| `/client` | TypeScript/React | Frontend | `packages/data-provider`, `packages/client` | React SPA |
| `/packages/client` | TypeScript | Frontend | `packages/data-provider` | Переиспользуемые UI компоненты |

### Граница рабочих пространств

- **Весь новый backend код** должен быть на **TypeScript** в `/packages/api`
- `/api` остаётся тонкой **JS оболочкой**, вызывающей `/packages/api`
- **Общая БД логика** → `/packages/data-schemas`
- **Frontend/backend общий API код** → `/packages/data-provider`

```bash
# Полная сборка через Turborepo (параллельная, кеширована)
```

## 📝 Стиль кода

### Принципы

#### 1. **Never-nesting** — Ранние возвраты

```javascript
// ❌ Плохо: глубокая вложенность
const processUser = (user) => {
  if (user) {
    if (user.active) {
      if (user.email) {
        sendEmail(user.email);
      }
    }
  }
};

// ✅ Хорошо: ранние возвраты
const processUser = (user) => {
  if (!user) return;
  if (!user.active) return;
  if (!user.email) return;
  sendEmail(user.email);
};
```

#### 2. **Functional first** — Чистые функции и неизменяемость

```javascript
// ❌ Плохо: императивный стиль
const filtered = [];
for (const item of items) {
  if (item.active) {
    filtered.push(item.name.toUpperCase());
  }
}

// ✅ Хорошо: функциональный стиль
const filtered = items
  .filter(item => item.active)
  .map(item => item.name.toUpperCase());
```

#### 3. **DRY** — Не повторяй себя

```javascript
// ❌ Плохо: дублирование логики валидации
const validateEmail = (email) => /^[^\s@]+@[^\s@]+\.[^\s@]+$/.test(email);
const checkEmail = (email) => /^[^\s@]+@[^\s@]+\.[^\s@]+$/.test(email);

// ✅ Хорошо: единая функция-валидатор
const isValidEmail = (email) => /^[^\s@]+@[^\s@]+\.[^\s@]+$/.test(email);
const validateEmail = isValidEmail;
const checkEmail = isValidEmail;
```

#### 4. **Type Safety** — Типизация без `any`

```typescript
// ❌ Плохо: использование any
const processData = (data: any) => {
  return data.value + 10;
};

// ✅ Хорошо: явная типизация
interface DataItem {
  value: number;
  name: string;
}

const processData = (data: DataItem): number => {
  return data.value + 10;
};
```

#### 5. **Performance** — Оптимизация итераций

```javascript
// ❌ Плохо: две итерации массива
const filtered = items.filter(item => item.active);
const uppercase = filtered.map(item => item.name.toUpperCase());

// ✅ Хорошо: одна итерация
const result = items
  .filter(item => item.active)
  .map(item => item.name.toUpperCase());

// ❌ Плохо: O(n²) сложность
const hasDuplicate = (arr) => {
  for (const a of arr) {
    for (const b of arr) {
      if (a === b && a !== b) return true;
    }
  }
};

// ✅ Хорошо: O(n) с Set
const hasDuplicate = (arr) => new Set(arr).size !== arr.length;
```

### JSDoc комментарии

```typescript
// ❌ Плохо: очевидный код с комментарием
// Увеличить счётчик на 1
counter++;

// ✅ Хорошо: самодокументирующийся код

// ✅ JSDoc для сложной логики
/**
 * Вычисляет стоимость сообщения на основе длины и модели

};
```

### Правила TypeScript

```typescript
// ❌ Не используйте unknown без обработки
const processData = (data: unknown) => {
  return (data as any).value; // ❌ Плохо!
};

// ✅ Явная типизация

  if (typeof data === 'object' && data !== null && 'value' in data) {
    return (data as { value: number }).value;
  }
  return null;
};

// ❌ Не дублируйте типы
interface User {

interface UserWithId {
  id: string;
  name: string;
  // ❌ Дублирование User
```

### CORS конфигурация

```javascript
// /api/server/middleware/cors.js
const allowedOrigins = [
  process.env.DOMAIN_CLIENT,
  'http://localhost:3090'
];

app.use(cors({
  origin: allowedOrigins,
  credentials: true,
  methods: ['GET', 'POST', 'PATCH', 'DELETE'],
  allowedHeaders: ['Content-Type', 'Authorization']
}));
```

### JWT токены

```env
JWT_SECRET=your-secret-key-change-in-production
JWT_REFRESH_SECRET=your-refresh-secret
JWT_EXPIRY=24h
```

### SSRF защита для Agent Actions

```yaml
# librechat.yaml
actions:
  allowedDomains:
    - 'swapi.dev'
    - 'api.example.com'
    # Localhost и приватные IP автоматически блокируются
```

## 🤝 Кодирование требования

При добавлении новых функций в LibreChat:

1. **Всегда пишите тесты** (unit + E2E)
2. **Соблюдайте стиль кода** (see AGENTS.md)
3. **Обновите документацию** и типы TypeScript
4. **Локализуйте текст** через `useLocalize()`
5. **Проверьте доступность** (ARIA labels, keyboard nav)
6. **Оптимизируйте производительность** (React Query, memo, lazy loading)
7. **Форматируйте код**: `npm run lint:fix && npm run format`
