## 📞 КОНТРОЛЬНЫЕ ВОПРОСЫ ДЛЯ ПРОВЕРКИ ПОНИМАНИЯ

### Вопрос 1: Почему нельзя переписывать Balance?
**Ответ:** Balance — оригинальный механизм LibreChat, используется везде. Любые изменения могут сломать другую функциональность.

### Вопрос 2: Что будет если забыть подключить payment route?
**Ответ:** Endpoint `/api/payment` вернет 404. Платежи работать не будут.

### Вопрос 3: Что будет если неправильный порядок middleware?
**Ответ:** Возможны проблемы:
- Нельзя получить req.builtEndpointOption если buildEndpointOption не был вызван
- Можно пробить баланс если checkBalance после спендинга
- Можно выбрать модель вне плана если checkSubscription не вызван

### Вопрос 4: Как защищена система от двойного платежа?
**Ответ:** Идемпотентность по externalPaymentId. Платеж с одним ID применится только один раз.

### Вопрос 5: Зачем кэш планов если есть БД?
**Ответ:** На каждый запрос не ходить в БД. Кэш на 60 сек достаточен для консистентности и производительности.
