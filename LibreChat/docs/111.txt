## 🎯 КЛЮЧЕВЫЕ ВЫВОДЫ

### Что произошло?

```
OAuth Flow:
1. Пользователь → Yandex (авторизация)
2. Yandex → LibreChat (возвращает profil + access_token)
3. LibreChat запрашивает профиль через API
4. ❌ ПРОБЛЕМА: Yandex не вернул email в профиле
5. ❌ Fallback срабатывает: генерируется yandex_ID@librechat.local
6. ❌ Пользователь создается с фиктивным email в БД
```

### Почему это происходит?

Есть несколько причин почему Yandex может не вернуть email:
- Пользователь не верифицировал email в Яндексе
- Приложение не запросило разрешение на доступ к email
- Старые API версии или изменения в Yandex
- Неправильная конфигурация scopes

### Почему это плохо?

1. **Невозможно связаться:** Email `yandex_2031223018@librechat.local` не существует
2. **GDPR нарушение:** Нельзя идентифицировать пользователей по email
3. **Дублирование:** Пользователь может создать новый аккаунт если забыл пароль
4. **Аудит:** Невозможно отследить кому принадлежит аккаунт

## 🔍 ТЕХНИЧЕСКАЯ СПРАВКА

### Проблемная строка

**Файл:** `api/server/controllers/auth/yandex.js:180`

```javascript
// ❌ БЫЛО (вызывает проблему)
const userEmail = yandexUser.default_email || yandexUser.emails?.[0] || `yandex_${yandexUser.id}@librechat.local`;
```

### Исправленная версия

```javascript
// ✅ СТАЛО (требует реальный email)
const userEmail = yandexUser.default_email || yandexUser.emails?.[0];

if (!userEmail) {
  logger.error(`[Yandex OAuth] No email in profile: ${yandexUser.id}`);
  return res.redirect(`${domains.client}/sign-in?error=yandex_no_email&provider=yandex`);
}
```

### Что проверяется?

1. `yandexUser.default_email` — основной email в профиле Yandex
2. `yandexUser.emails?.[0]` — первый email из массива
3. Если оба undefined → ошибка вместо fallback на @librechat.local
