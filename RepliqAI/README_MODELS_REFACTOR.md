## ❓ ЧТО ИЗМЕНИЛОСЬ?

### В двух словах:
**Селектор моделей теперь загружает модели из backend API вместо hardcoded списка, фильтруя их по плану пользователя.**

### ДО:
```
Администратор видит → 16 моделей в админке
Пользователь видит → все 16 моделей в селекторе (без фильтрации)
Отправляет сообщение → 403 ошибка если модель не в плане ❌
```

### ПОСЛЕ:
```
Администратор видит → 16 моделей в админке (без изменений)
Пользователь видит → ТОЛЬКО модели из его плана ✅
Отправляет сообщение → успешно, без ошибок ✅
```

## 📝 ЧТО НУЖНО ЗНАТЬ?

### 1. Для разработчиков

**Где находится реализация:**
- `client/src/components/Chat/Menus/Endpoints/ModelSelectorContext.tsx`
- Строки добавлено: ~50
- Основное изменение: добавлен `useQuery` для `/api/models/allowed`

**Что это значит:**
- Селектор теперь делает запрос на `GET /api/models/allowed`
- Backend фильтрует модели по плану пользователя
- Результат кэшируется на 60 секунд

**Как тестировать:**
```bash
# 1. Открыть DevTools → Network
# 2. Открыть селектор моделей
# 3. Увидеть запрос: GET /api/models/allowed
# 4. Проверить ответ: содержит только разрешенные модели
# 5. Выбрать модель
# 6. Отправить сообщение
# 7. ✅ Должно отправиться без ошибок
```

### 2. Для администраторов

**Как управлять доступом к моделям:**
1. Открыть `/admin/settings` (вкладка Settings)
2. Найти карточку тариф'а (Free, Pro, Business)
3. Выбрать чекбоксы с нужными моделями
4. Нажать "Сохранить"
5. **ГОТОВО** — при обновлении страницы пользователи получат новые модели

**Кэширование:**
- Кэш инвалидируется на backend сразу при сохранении
- На frontend обновляется при следующем запросе (max 60 сек)

## 🔍 БЫСТРАЯ ДИАГНОСТИКА

### "Селектор показывает пусто"
- ✅ Нормально, если план`=`free с пустым allowedModels
- 📝 Проверить в админке: какие модели разрешены для free плана?
- 🔧 Добавить модели через `/admin/settings`

### "Вижу 403 ошибку при отправке"
- ⚠️ Это может быть если модель попала в payload но не в плане
- 📝 Проверить Plan.allowedModels в БД: `db.plans.findOne({planId:'pro'})`
- 🔧 Добавить модель в админке

### "Селектор загружается медленно"
- ✅ Нормально при первом открытии (~200-500ms)
- ✅ Будет быстрым при повторном открытии (<1ms из кэша)

### "/api/models/allowed возвращает ошибку"
- 🔴 Критично — нужен fallback
- 🔧 Проверить: пользователь авторизован? Токен не истек?
- ✅ Fallback автоматически использует startupConfig.modelSpecs
