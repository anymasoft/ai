# 🔓 КАК РАЗБЛОКИРОВАТЬ СВОЮ УЧЕТНУЮ ЗАПИСЬ

## 🔥 ПРОБЛЕМА
Вы получили сообщение:
> "Ваша учетная запись была временно заблокирована в связи с нарушениями нашего сервиса."

## 🎯 РЕШЕНИЕ

### Что произошло?
Механизм защиты от нарушений (`BAN_VIOLATIONS`) был **включен** в конфигурации.
Когда вы пытались использовать `claude-haiku-4-5` с ошибкой `illegal_model_request`, система накопила нарушения и заблокировала вашу учетную запись.

### ✅ Как это исправлено?

**Создан новый файл `.env`** с:
```
BAN_VIOLATIONS=false
LOGIN_VIOLATION_SCORE=0
REGISTRATION_VIOLATION_SCORE=0
CONCURRENT_VIOLATION_SCORE=0
MESSAGE_VIOLATION_SCORE=0
NON_BROWSER_VIOLATION_SCORE=0
```

Все механизмы блокировки **отключены** для локальной разработки.

### 🚀 ЧТО ДЕЛАТЬ ТЕПЕРЬ

#### Шаг 1: Подтвердите что `.env` создан
```bash
cat /home/user/ai/LibreChat/.env | grep BAN_VIOLATIONS
# Должна выводить: BAN_VIOLATIONS=false
```

#### Шаг 2: Перезагрузите сервер
```bash
# Закройте текущий сервер (Ctrl+C)
# Затем запустите заново:
npm run backend:dev
```

#### Шаг 3: Очистите данные сессии (опционально)

Если после перезагрузки вы всё ещё видите сообщение об блокировке:

**Способ 1: Используйте приватный режим браузера**
```
Откройте новый приватный/incognito окно
Зайдите на http://localhost:3080
Создайте новую учетную запись или используйте другой браузер
```

**Способ 2: Очистите куки браузера**
```
F12 → Application (Applications) → Cookies → http://localhost:3080
Удалите все куки и refresh
```

**Способ 3: Удалите куки через консоль браузера**
```javascript
document.cookie.split(";").forEach(function(c) {
  document.cookie = c
    .replace(/^ +/, "")
    .replace(/=.*/, "=;expires=" + new Date().toUTCString() + ";path=/");
});
```

#### Шаг 4: Зайдите в систему
```
http://localhost:3080
Используйте свои учетные данные
```

## ⚠️ ПОЧЕМУ ВАС ЗАБЛОКИРОВАЛИ?

Механизм защиты срабатывает когда:
- 🔴 Используется недоступная модель (для вас это была `claude-haiku-4-5` до исправления)
- 🔴 Слишком много ошибок подряд
- 🔴 Подозрительная активность

Это хорошая защита для production-сервера, но для разработки она мешает.

## ✨ ЧТО ИЗМЕНИЛОСЬ

| До | После |
|----|-------|
| `BAN_VIOLATIONS=true` | `BAN_VIOLATIONS=false` |
| Блокировка включена | Блокировка отключена |
| Риск бана при ошибках | Никаких ограничений |
