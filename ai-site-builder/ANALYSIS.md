# AI Site Builder - Полный анализ проекта и карта монетизации

---

## 1. ТЕХНИЧЕСКОЕ РЕЗЮМЕ ПРОЕКТА

### Архитектура
Full-stack монорепозиторий с раздельными клиентом и сервером.

| Слой | Технология |
|------|-----------|
| Frontend | React 19 + TypeScript + Vite + Tailwind CSS v4 + shadcn/ui |
| Backend | Express 5 + TypeScript + SQLite (better-sqlite3) |
| AI-движок | OpenAI GPT-4o-mini (2 вызова на генерацию: улучшение промпта + генерация HTML) |
| Аутентификация | better-auth (сейчас dev-режим с хардкод-пользователем) |
| Оплата | Stripe (заглушка, env-переменные подготовлены, но интеграция не реализована) |

### База данных (SQLite)
- **users** — id, email, name, credits (стартовые 20)
- **projects** — id, user_id, name, initial_prompt, current_code, is_published
- **conversations** — история чата по каждому проекту (user/assistant)
- **versions** — полный код каждой версии проекта с таймстампами

### Что проект ФАКТИЧЕСКИ умеет делать (уже реализовано)

1. **Генерация полного HTML-сайта из текстового описания** — пользователь вводит промпт на естественном языке, AI генерирует готовый одностраничный HTML с Tailwind CSS
2. **Улучшение промптов** — каждый пользовательский запрос сначала проходит через "prompt enhancement" (дизайн-детали, секции, UX, адаптивность)
3. **Чат-ревизии** — итерационные правки через текстовые команды ("сделай кнопку красной", "добавь секцию отзывов")
4. **Визуальный редактор элементов** — кликаешь на элемент в iframe → редактируешь текст, классы, отступы, цвета
5. **Система версий** — полная история изменений с откатом на любую версию
6. **Превью в 3 разрешениях** — десктоп, планшет, телефон
7. **Публикация** — переключатель publish/unpublish, витрина опубликованных сайтов (Community)
8. **Скачивание HTML** — экспорт готового файла index.html
9. **Кредитная система** — 5 кредитов за создание/ревизию, тарифы Basic ($5/100 кр.), Pro ($19/400 кр.), Enterprise ($49/1000 кр.)
10. **Авторизация с управлением аккаунтом** — страницы входа/регистрации/настроек

### Что ЛЕГКО добавить (минимальные доработки)

- **Stripe-оплата** — переменные окружения уже подготовлены, нужен webhook handler (~2-4 часа работы)
- **Продакшн-аутентификация** — better-auth уже интегрирован, нужно убрать dev-режим
- **Библиотека шаблонов** — предустановленные промпты по нишам (ресторан, салон, юрист и т.д.)
- **Кастомный домен** — HTML статичный, легко деплоить на любой хостинг
- **Мультиязычность генерации** — достаточно изменить системный промпт
- **White-label** — убрать брендинг, дать возможность подставлять свой
- **Пакетная генерация через API** — бэкенд уже принимает промпты по API
- **Email-шаблоны** — изменить системный промпт для генерации email-совместимого HTML
- **Многостраничные сайты** — расширить pipeline генерации

## 2. АРХИТЕКТУРНЫЕ ОГРАНИЧЕНИЯ

| Ограничение | Влияние |
|------------|---------|
| **Только одностраничные сайты** | Нет роутинга, навигации между страницами |
| **Нет хостинга** | Генерирует HTML, но не хостит — нужен отдельный деплой |
| **GPT-4o-mini** | Дешёвый, но менее качественный, чем GPT-4o. Сложные сайты могут быть слабыми |
| **Только placeholder-изображения** | Нет загрузки реальных фото, только placehold.co |
| **Формы не работают** | Визуально есть, но бэкенда для обработки форм нет |
| **Нет SEO-инструментов** | Мета-теги генерируются, но нет schema.org, sitemap, analytics |
| **Dev-режим auth** | Для продакшна нужно доделать аутентификацию |
| **Stripe не подключён** | Оплата — заглушка, кредиты просто начисляются |

**Важный вывод:** Технология лучше всего подходит для **быстрой генерации лендингов и одностраничников** — это её сильная сторона. Пытаться делать из неё конструктор полноценных сайтов — долго и бессмысленно при наличии Wix/Tilda.

## 3. КАРТА ВОЗМОЖНОСТЕЙ — ЧТО ТЕХНОЛОГИЯ РЕАЛЬНО УМЕЕТ

```
ВХОД: Текстовое описание на любом языке
  ↓
ПРОЦЕСС: Улучшение промпта → Генерация HTML+Tailwind → Итерационные правки
  ↓
ВЫХОД: Готовый HTML-файл (одностраничный, адаптивный, с JS-интерактивом)
```

**Ключевая ценность:** За 30-60 секунд создаётся визуально приличный лендинг, который раньше стоил $200-2000 и занимал дни/недели.

**Задачи бизнеса, которые это решает:**
- Быстрое создание посадочных страниц для рекламных кампаний
- Прототипирование идей и продуктов
- Создание веб-присутствия для офлайн-бизнесов
- A/B тестирование разных вариантов лендингов
- Генерация промо-страниц для мероприятий
- Создание портфолио и визиток

**H) Ожидаемый чек:** 5 000–25 000 руб. за лендинг ($50–250). Хостинг/поддержка: 1 000–3 000 руб./мес.

## 5. ТОП-3 ИДЕИ С МАКСИМАЛЬНОЙ ВЕРОЯТНОСТЬЮ БЫСТРЫХ ПЕРВЫХ ОПЛАТ

### #1: Продажа лендингов на фриланс-площадках (ИДЕЯ 2)
**Почему первое место:** Ноль доработок. Ноль затрат на маркетинг. Клиенты УЖЕ ИЩУТ и УЖЕ ГОТОВЫ ПЛАТИТЬ. Начинаешь сегодня.

### #2: "Сайт за час" для локальных бизнесов (ИДЕЯ 1)
**Почему второе место:** Ноль доработок в продукте. Но нужно выйти из дома и начать продавать лично. Чек выше, чем на фрилансе.

### #3: SaaS для арбитражников / маркетологов (ИДЕЯ 3)
**Почему третье место:** Требует минимальных доработок (Stripe + auth + деплой = 1 день работы), но после запуска даёт повторяющийся доход (MRR).

## 6. ДЕТАЛЬНЫЙ ПЛАН MVP ДЛЯ КАЖДОЙ ИЗ ТОП-3

---

### MVP #1: Фриланс-конвейер лендингов

**Самый простой MVP:**
- Развернуть ai-site-builder локально (уже работает)
- Создать профили на Kwork, Fiverr, Upwork, FL.ru
- Сгенерировать 10-15 примеров лендингов для разных ниш (ресторан, стоматология, фитнес, юрист, ремонт)
- Сделать скриншоты для портфолио
- Опубликовать гиги/услуги с ценой $50-150

**Какие фичи НЕ делать:**
- НЕ делать Stripe
- НЕ делать продакшн-аутентификацию
- НЕ деплоить как SaaS
- НЕ делать шаблоны (промпты пишешь руками)
- НЕ делать мультистраничность

**Первый шаг:**
1. Запустить сервер локально (`npm run server` + `npm run dev`)
2. Сгенерировать 5 красивых лендингов для портфолио
3. Создать профиль на Kwork с услугой "Лендинг за 24 часа"
4. Выставить цену 3 000 - 5 000 руб.
5. Ждать первый заказ (обычно 1-7 дней на Kwork)

---

### MVP #2: "Сайт за час" для локального бизнеса

**Самый простой MVP:**
- ai-site-builder на ноутбуке
- Список из 20-30 локальных бизнесов без сайтов (находятся через Google Maps / 2GIS — ищи компании БЕЗ ссылки на сайт)
- 5-7 готовых примеров по нишам (кафе, салон, автосервис)
- Простой оффер: "Покажу готовый сайт для вашего бизнеса прямо сейчас. Бесплатно. Если понравится — обсудим цену"

**Какие фичи НЕ делать:**
- НЕ делать сложный хостинг (используй Netlify Drop / GitHub Pages — бесплатно)
- НЕ делать формы обработки заявок (хватит простого mailto: или WhatsApp-ссылки)
- НЕ делать CMS / админку
- НЕ делать SEO-оптимизацию (это upsell на будущее)

**Первый шаг:**
1. Открыть Google Maps / 2GIS
2. Найти 10 бизнесов рядом с тобой, у которых НЕТ сайта
3. Сгенерировать для каждого лендинг (используя инфу из карт: название, адрес, телефон, часы работы, фото)
4. Прийти к ним лично со словами: "Я сделал вам сайт. Посмотрите. Если нравится — 10 тысяч рублей и он ваш"
5. Первая продажа — вопрос 1-3 дней обхода

### MVP #3: SaaS для маркетологов

**Самый простой MVP:**
- Подключить Stripe (webhook + checkout session) — код для env-переменных уже есть
- Убрать dev-режим аутентификации (активировать better-auth в продакшне)
- Деплой на Railway / Render / VPS
- Добавить 5 промпт-шаблонов: Sales Page, Squeeze Page, Webinar Page, Thank You Page, Coming Soon
- Поставить лимит: 3 бесплатные генерации, потом платная подписка

**Какие фичи НЕ делать:**
- НЕ делать собственный хостинг сгенерированных сайтов (пусть скачивают HTML)
- НЕ делать A/B тестирование (это v2)
- НЕ делать интеграции с рекламными платформами
- НЕ делать custom domains
- НЕ делать аналитику
- НЕ делать мультистраничность

**Первый шаг:**
1. Подключить Stripe (4 часа)
2. Активировать better-auth (2 часа)
3. Деплой на Railway (1 час)
4. Написать пост в 5-10 Telegram-каналов по арбитражу / маркетингу
5. Запустить на Product Hunt
6. Первые платящие пользователи — в течение 1-2 недель

## 7. КРИТИЧЕСКАЯ ОЦЕНКА

**Самый честный совет:** Начни с MVP #1 (фриланс). Это единственный вариант, где деньги можно получить СЕГОДНЯ, не написав ни строчки нового кода. Генерируешь лендинг, продаёшь результат. Всё.

Параллельно готовь MVP #3 (SaaS), потому что это единственный из трёх вариантов, который масштабируется без твоего линейного времени.

MVP #2 (обход бизнесов) — высокий чек, но требует личных продаж. Это хорошо для первых 10-20 клиентов, но не масштабируется.

**Технология лучше подходит не для "конструктора сайтов", а для:**
- Инструмента быстрого прототипирования (генерация MVP-макетов за секунды)
- Конвейера лендингов (массовое производство одностраничников)
- AI-ассистента веб-разработчика (ускорение рутинной работы)

Не пытайся конкурировать с Wix/Tilda/Webflow — они решают задачу "полноценного сайта". Твоя ниша — **скорость и простота для одностраничников**.
