# CSV Устойчивость к ошибкам

## Проблема
Иногда при аварийной записи CSV файл повреждается, что вызывает ошибку:
```
_csv.Error: line contains NUL
```

Это приводит к:
- Краху парсера
- Потере данных
- Прерыванию процесса

## Решение

### 1. Очистка NUL символов
Перед чтением CSV файла все NUL символы (`\x00`) удаляются:
```python
content = content.replace('\x00', '')
```

### 2. Graceful Degradation
При ошибке чтения строка пропускается, парсер продолжает работу:
```python
try:
    for row in csv_reader:
        # обработка строки
except csv.Error as e:
    logger.warning(f"Ошибка чтения: {e}. Пропуск строки.")
    continue
```

### 3. Режим Append (дописывание)
Если CSV файл уже существует, используется режим `append` вместо перезаписи:
```python
if file_exists:
    self._file = self._open_file(self._file_path, 'a')  # mode='a'
```

**Результат:** При повторном запуске парсер дописывает данные, не теряя уже собранные.

### 4. Безопасное закрытие
Файл закрывается с `flush()` для гарантии записи данных:
```python
self._file.flush()
self._file.close()
```

### 5. Откат временных файлов
При критических ошибках временные файлы удаляются:
```python
if os.path.exists(tmp_csv_name):
    os.remove(tmp_csv_name)
```

## Обработанные методы

### `_writerow(row)`
- Перехватывает `UnicodeEncodeError`, `csv.Error`, и общие исключения
- Логирует ошибку и продолжает работу (не выбрасывает исключение)
- Обновлено: добавлена безопасность к `row.get('name', 'N/A')`

### `_remove_empty_columns()`
- Очищает CSV от NUL перед чтением
- Try/except для подсчёта пустых колонок
- Try/except для переписи данных
- Логирует количество успешно обработанных vs пропущенных строк
- При ошибке откатывает временный файл

### `_remove_duplicates()`
- Очищает CSV от NUL перед обработкой
- Try/except для каждой строки
- Логирует статистику: кол-во написанных vs пропущенных
- При критической ошибке откатывает временный файл

### `__enter__()` / `__exit__()`
- Обнаруживает существующий CSV файл
- Если существует → режим append + подсчёт строк
- Если нет → режим write + запись заголовка
- `__exit__` безопасно закрывает файл и обрабатывает пост-обработку с try/except

## Покрытие ошибок

| Тип ошибки | Обработка | Результат |
|-----------|----------|----------|
| `_csv.Error: line contains NUL` | Очистка `\x00` перед чтением | ✓ Файл читается без ошибок |
| UnicodeEncodeError | Try/except в `_writerow()` | ✓ Строка пропускается, парсер продолжает |
| csv.Error при чтении | Try/except в циклах | ✓ Битая строка пропускается |
| Критическая ошибка пост-обработки | Try/except + откат временного файла | ✓ Оригинальный CSV неизменён |
| Аварийное завершение парсера | Режим append на перезапуск | ✓ Данные дописываются, не теряются |

## Технические детали

### Функция очистки
```python
def _clean_csv_content(content: str) -> str:
    """Очистить CSV контент от NUL символов"""
    content = content.replace('\x00', '')
    return content
```

### Подсчёт строк при append
```python
def _count_existing_rows(self) -> int:
    """Подсчитать существующие строки (для корректного счётчика)"""
    try:
        with open(file_path) as f:
            content = _clean_csv_content(f.read())
        row_count = sum(1 for i, line in enumerate(content.split('\n'))
                       if i > 0 and line.strip())
        return row_count
    except Exception:
        return 0
```
