# Оценка коммерческого потенциала: VideoSOS

**Дата анализа:** 2026-02-15
**Репозиторий:** https://github.com/timoncool/videosos.git
**Демо:** https://videosos.vercel.app/
**Лицензия:** MIT (полная коммерческая свобода)

---

## 1. Что это за проект

VideoSOS — open-source AI-видеоредактор в браузере. Позволяет генерировать видео, изображения, музыку и озвучку через 100+ AI-моделей (Google Veo, Kling, FLUX, Imagen и др.). Все данные хранятся локально в IndexedDB, API-ключи пользователь вводит свои.

**Стек:** Next.js 14, React 18, TypeScript (strict), Zustand, React Query, Remotion, FFmpeg.wasm, Tailwind CSS, Radix UI.

**Провайдеры AI:** fal.ai (100+ моделей), Runware.ai (122 модели).

**GitHub:** ~1.1k звёзд, 454 форка, 654 коммита, активная разработка.

---

## 2. Краткий ответ: можно ли сделать коммерческий сервис?

**Да, можно. Но "быстро" — это относительно.**

Проект даёт отличную основу для UI/UX видеоредактора, но для коммерческого сервиса нужно добавить серверную инфраструктуру, которой сейчас фактически нет.

## 5. Варианты монетизации

### Вариант A: "Прокси-модель" (самый быстрый запуск)

Пользователи платят вам, вы оплачиваете API провайдерам с наценкой.

- Добавить: Auth + Stripe + серверный proxy для API + тарифы
- Маржа: 30-50% наценка на стоимость генерации
- Плюс: Пользователям не нужно разбираться с API-ключами
- Минус: Высокие расходы на API при масштабировании

### Вариант B: "Подписочная модель" (SaaS)

Месячная подписка с включёнными кредитами.

- Free: 50 генераций/месяц
- Pro ($19-29/мес): 500 генераций + облачное хранение
- Business ($49-99/мес): безлимит + приоритет + API доступ
- Плюс: Предсказуемый доход
- Минус: Нужно больше серверной инфраструктуры

### Вариант C: "Pay-as-you-go" (кредитная модель)

Пользователи покупают кредиты, тратят на генерации.

- Пакеты: $5 / $20 / $50 / $100
- Плюс: Прозрачное ценообразование, система расчёта стоимости уже есть
- Минус: Менее предсказуемый доход

## 6. Оценка объёма работ для коммерциализации

### Минимальный MVP (Вариант A):

1. **Аутентификация** — NextAuth.js или Clerk
2. **БД для пользователей** — PostgreSQL + Prisma
3. **Серверный proxy для AI** — перенести API-вызовы на сервер
4. **Stripe интеграция** — простая оплата по факту
5. **Rate limiting** — ограничения для бесплатного тарифа
6. **Деплой** — Vercel Pro / Railway / собственный сервер

### Полноценный SaaS (Вариант B):

Всё из MVP + облачное хранение проектов + admin-панель + email + мониторинг + тесты.

Это не "быстро взять и запустить", но и не "писать с нуля". Основная ценность — готовый видеоредактор с timeline, который работает в браузере.

Главный вопрос не технический, а бизнесовый: **чем вы будете отличаться от Runway, Pika и десятков других AI-видеосервисов?** Без чёткого ответа на этот вопрос техническая реализация не имеет значения.

## 9. СТРАТЕГИЯ: ПРОКСИ-СЕРВИС ИЗ ГРУЗИИ ДЛЯ РУССКОЯЗЫЧНОЙ АУДИТОРИИ

### 9.1. Бизнес-модель

**Оператор:** Грузия (нет санкционных ограничений)
**Серверы:** Европа
**AI-провайдеры:** fal.ai + Runware (ключи уже есть)
**Целевая аудитория:** русскоязычные пользователи, которые не могут напрямую оплатить западные AI-сервисы
**Суть:** Прокси-сервис — пользователи платят в рублях через ЮKassa, сервис проксирует запросы к fal.ai/Runware через собственные API-ключи

### 9.2. Почему это работает

Ситуация в РФ для конечных пользователей:
- **Runway, Pika, Sora** — заблокированы / нельзя оплатить
- **fal.ai, Runware** — нельзя оплатить (нет международных карт)
- **Оплата в долларах/евро** — невозможна для большинства
- **Прямых конкурентов (AI-видеоредактор + оплата в рублях)** — нет

Ваш сервис решает именно эту проблему: **даёт доступ к 100+ AI-моделям с оплатой в рублях**.

### 9.3. Архитектура сервиса

```
[Пользователь] → [Ваш сервер (Европа)] → [fal.ai / Runware]
     ↑                    ↑
  ЮKassa            Ваши API-ключи
  (рубли)          ($ на вашу карту)
```

Ключевое изменение относительно оригинального VideoSOS:
- **Сейчас:** пользователь вводит СВОИ API-ключи в localStorage → запросы идут напрямую
- **Нужно:** пользователь платит ВАМ → запросы идут через ВАШ серверный proxy с ВАШИМИ ключами

### 9.4. Что нужно доработать (конкретный план)

AI-провайдеры (fal.ai + Runware) **остаются как есть** — менять не нужно. Все 100+ моделей работают.

| # | Задача | Сложность | Срок | Описание |
|---|---|---|---|---|
| 1 | **Серверный API-proxy** | Средняя | 3-5 дней | Перенести вызовы fal.ai/Runware с клиента на серверные API Routes. Убрать localStorage-ключи, использовать свои ключи на сервере |
| 2 | **Аутентификация** | Средняя | 2-3 дня | NextAuth.js с email/пароль. Опционально: Yandex ID, VK ID, Telegram Login |
| 3 | **БД для пользователей** | Средняя | 2-3 дня | PostgreSQL + Prisma. Таблицы: users, subscriptions, credits, generations (лог) |
| 4 | **ЮKassa интеграция** | Средняя | 3-5 дней | Оплата подписок + пакетов кредитов. REST API + Checkout Widget + вебхуки |
| 5 | **Система кредитов/лимитов** | Средняя | 2-3 дня | Middleware: проверка баланса перед генерацией, списание после успеха, rate limiting |
| 6 | **Деплой** | Низкая | 1 день | VPS в Европе (Hetzner/DigitalOcean) или Vercel Pro + managed PostgreSQL |

**Итого: ~2-3 недели** на MVP (один разработчик, полный рабочий день)

### 9.5. Детали серверного proxy

Сейчас в VideoSOS вызовы AI идут так (клиент):
```typescript
// src/lib/fal.ts — ТЕКУЩАЯ РЕАЛИЗАЦИЯ
const fal = createFalClient({
  credentials: () => localStorage.getItem('falKey') // ← ключ пользователя
});
const result = await fal.subscribe('fal-ai/flux-pro/kontext', { input: {...} });
```

Нужно переделать на серверный proxy:
```typescript
// src/app/api/generate/route.ts — НОВАЯ РЕАЛИЗАЦИЯ
export async function POST(req: Request) {
  const session = await getServerSession(authOptions);
  if (!session) return Response.json({ error: 'Unauthorized' }, { status: 401 });

  const user = await db.user.findUnique({ where: { id: session.user.id } });
  if (user.credits <= 0) return Response.json({ error: 'No credits' }, { status: 402 });

  const { model, input } = await req.json();
  const cost = calculateCost(model, input); // система расчёта уже есть в проекте!

  const fal = createFalClient({ credentials: () => process.env.FAL_API_KEY }); // ← ВАШ ключ
  const result = await fal.subscribe(model, { input });

  await db.user.update({ where: { id: user.id }, data: { credits: { decrement: cost } } });
  await db.generation.create({ data: { userId: user.id, model, cost, status: 'success' } });

  return Response.json(result);
}
```

Аналогично для Runware. Фронтенд-вызовы заменяются с прямых на `/api/generate`.

### 9.6. Интеграция ЮKassa

**Комиссии:**

| Способ оплаты | Комиссия |
|---|---|
| Банковские карты (Мир, Visa, MC) | ~3.42% (2.8% + НДС на комиссию) |
| СБП (Система быстрых платежей) | ~1.5% |
| SberPay / T-Pay / Mir Pay | ~1.5% |
| ЮMoney кошелёк | 3.0% |

**Интеграция с Next.js:**
- Community TypeScript SDK: `yookassa-ts-sdk`
- REST API: `https://api.yookassa.ru/v3/` с Basic Auth
- Встраиваемый Checkout Widget для формы оплаты на странице
- Вебхуки для подтверждения платежа

**Важно:** ЮKassa работает только с юрлицами/ИП, зарегистрированными в РФ. Варианты:
- Зарегистрировать ИП/ООО в РФ (или партнёр в РФ)
- Альтернатива: **Robokassa** (работает с иностранными компаниями)
- Альтернатива: **Cryptomus / криптоплатежи** (USDT → без привязки к юрисдикции)

### 9.7. Тарифные планы (в рублях)

**Модель: Подписка + дополнительные кредиты**

| Тариф | Цена | Включено |
|---|---|---|
| **Бесплатный** | 0 ₽ | 3 видео / 10 картинок (ознакомиться) |
| **Старт** | 990 ₽/мес | 50 видео / 200 картинок |
| **Про** | 2 490 ₽/мес | 200 видео / 1000 картинок + приоритет |
| **Бизнес** | 6 990 ₽/мес | 500 видео / безлимит картинок |

Дополнительные кредиты: 299 ₽ / 999 ₽ / 2 999 ₽

**Экономика (пример для тарифа "Старт" — 990 ₽/мес):**
- Себестоимость 50 видео через fal.ai: ~$10-22 (по $0.20-0.45 за видео)
- 990 ₽ ≈ $10-11 по курсу
- Маржа: **зависит от моделей**, которые выбирает пользователь. На дешёвых моделях (FLUX) — хорошая. На дорогих (Veo 3.1 по $0.45) — тонкая
- **Решение:** ограничить дорогие модели для старших тарифов или считать по кредитам (1 кредит ≠ 1 генерация, зависит от модели)
