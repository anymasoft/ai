# SaaS-стартап с бюджетом $1000: стратегия быстрых первых оплат

## 1. Стратегия выбора ниши

Логика отбора строится на трёх фильтрах:

**Фильтр 1: Боль, за которую уже платят.** Не изобретаем новую категорию. Берём существующую проблему, за решение которой люди уже отдают деньги конкурентам — но конкуренты либо дорогие, либо перегружены функциями, либо неудобны для конкретного сегмента.

**Фильтр 2: B2B-микросегмент.** B2C требует массового трафика и долгой воронки. B2B-клиенты принимают решение быстрее, платят больше, и их можно найти поштучно через холодный аутрич. Идеальный сегмент — малый бизнес (5–50 сотрудников), фрилансеры, агентства, соло-консультанты.

**Фильтр 3: "Скучные" задачи.** Чем скучнее задача (выставление счетов, отчёты, мониторинг, шаблоны), тем выше готовность заплатить за автоматизацию и тем ниже конкуренция со стороны стартапов, которые гонятся за хайпом.

**Антипаттерны — чего избегать:**
- Маркетплейсы (проблема курицы и яйца)
- Продукты, требующие интеграций с крупными платформами на старте
- Всё, где нужна модерация пользовательского контента
- Продукты для разработчиков (крайне высокая конкуренция, привычка к бесплатному)

## 2. Пять конкретных идей

---

### Идея 1: Генератор и трекер коммерческих предложений для малого бизнеса

**Целевая аудитория:** Небольшие B2B-компании (веб-студии, маркетинговые агентства, IT-аутсорс, консалтинг) — 1–20 человек.

**Проблема:** Каждый раз коммерческое предложение (КП) собирается вручную в Google Docs или Word. Нет аналитики: неизвестно, открыл ли клиент документ, сколько времени смотрел, на каком слайде закрыл. Нет шаблонов, нет версионирования, нет напоминаний о follow-up.

**Почему заплатят быстро:** Прямая связь с деньгами. КП — это последний шаг перед сделкой. Любое улучшение конверсии КП напрямую увеличивает выручку клиента. Это не "nice to have", а инструмент продаж.

**Минимальный MVP:**
- Веб-редактор КП с 5–10 отраслевыми шаблонами
- Генерация ссылки для отправки клиенту (вместо PDF)
- Трекинг: открытие, время просмотра, повторные визиты
- Email-уведомления владельцу: "Ваш клиент открыл КП"
- Простая панель со списком всех КП и их статусов

**Стек технологий:**
- Next.js (фронт + API)
- PostgreSQL (Supabase — бесплатный tier)
- Vercel (хостинг, бесплатный tier)
- Resend или Postmark (email-уведомления)
- Stripe или Lemon Squeezy (оплата)

**Как получить первых 20 клиентов:**
- Холодный аутрич в LinkedIn/Telegram-чаты для агентств и фрилансеров (0$)
- Посты в профильных сообществах: vc.ru, Habr, тематические Telegram-каналы (0$)
- Предложить 10 агентствам бесплатный месяц в обмен на обратную связь, потом конвертировать в платящих
- Product Hunt запуск (0$)

**Финансовая модель:**
| Параметр | Значение |
|---|---|
| Цена | $29/мес (базовый), $59/мес (про: неограниченные КП, кастомный домен, branding) |
| Себестоимость на клиента | ~$1–2/мес (хостинг, email) |
| Маржа | ~95% |
| Для $3k MRR | ~70 клиентов на базовом плане |
| Для $5k MRR | ~115 клиентов на базовом или ~85 на миксе |

### Идея 2: Мониторинг аптайма и статус-пейдж для малого бизнеса

**Целевая аудитория:** SaaS-компании на ранней стадии, веб-студии, которые поддерживают клиентские сайты, фрилансеры-разработчики с пулом клиентов.

**Проблема:** Крупные решения (Datadog, PagerDuty) стоят сотни долларов и перегружены функциями. Мелкие конкуренты (UptimeRobot, BetterStack) либо ограничены в бесплатном плане, либо быстро дорожают. Нужна простая штука: пинг URL каждые N минут + красивая статус-страница + алерты в Telegram/Slack/email.

**Почему заплатят быстро:** Падение сайта = потеря денег и репутации. Это страховка — люди платят за неё регулярно и не отписываются, пока бизнес работает. Низкий churn.

**Минимальный MVP:**
- HTTP(S) проверки с интервалом 1–5 минут
- Алерты: email + Telegram (webhook)
- Публичная статус-страница (кастомный поддомен)
- Дашборд с историей аптайма за 30/90 дней

**Стек технологий:**
- Go или Node.js (бэкенд-воркеры для пингов)
- Redis + PostgreSQL (хранение результатов)
- Next.js (дашборд)
- Cron-job сервис или собственный scheduler
- Hetzner VPS ($5–10/мес) для воркеров
- Vercel/Cloudflare Pages для фронта

**Как получить первых 20 клиентов:**
- Посты в IndieHackers, Reddit (r/SaaS, r/webdev), Telegram-чаты разработчиков
- Интеграция с Telegram — выложить бота, который сам является точкой входа
- Холодный аутрич веб-студиям: "Вы мониторите аптайм сайтов ваших клиентов?"
- Демо-сравнение с конкурентами по цене (подчеркнуть, что дешевле)

**Финансовая модель:**
| Параметр | Значение |
|---|---|
| Цена | $9/мес (10 мониторов), $19/мес (50), $39/мес (200) |
| Себестоимость | ~$2–5/мес на инфраструктуру при <100 клиентах |
| Маржа | ~80–90% |
| Для $3k MRR | ~200 клиентов на среднем плане |
| Для $5k MRR | ~330 клиентов |

**Риск:** Высокая конкуренция. Дифференциация только через цену и UX — это слабая позиция. Масштабирование требует инфраструктуры.

### Идея 3: Сервис автоматических отчётов для клиентов агентств

**Целевая аудитория:** Маркетинговые агентства, SMM-специалисты, SEO-фрилансеры — те, кто ежемесячно отправляют клиентам отчёты о проделанной работе.

**Проблема:** Каждый месяц агентство тратит 2–5 часов на одного клиента, вручную собирая скриншоты из Google Analytics, рекламных кабинетов, соцсетей. Потом вставляет всё в Google Slides, добавляет комментарии, отправляет PDF. При 10–20 клиентах это превращается в ад.

**Почему заплатят быстро:** Экономия 20–40 часов в месяц — это конкретная, считаемая цифра. Агентства хорошо считают unit-экономику и понимают, что $49/мес за автоматизацию дешевле, чем час работы junior-маркетолога.

**Минимальный MVP:**
- Подключение Google Analytics и Яндекс.Метрики через API
- Автоматическая генерация PDF/HTML-отчёта по расписанию (раз в неделю/месяц)
- 3–5 шаблонов отчётов
- White-label: логотип и цвета агентства
- Автоотправка клиенту на email

**Стек технологий:**
- Next.js + tRPC
- PostgreSQL (Supabase)
- Google Analytics Data API, Яндекс.Метрика API
- Puppeteer или @react-pdf/renderer (генерация PDF)
- Bull/BullMQ (очередь задач для генерации по расписанию)
- Vercel + один VPS для воркера

**Как получить первых 20 клиентов:**
- Telegram-чаты и форумы для маркетологов (их десятки, активные)
- Посты на vc.ru с кейсом "Как мы автоматизировали отчётность"
- Прямые сообщения агентствам в Instagram/LinkedIn
- Партнёрство с 2–3 блогерами-маркетологами (бесплатный аккаунт за упоминание)

**Финансовая модель:**
| Параметр | Значение |
|---|---|
| Цена | $49/мес (до 10 клиентов), $99/мес (до 30 клиентов) |
| Себестоимость | ~$3–5/мес |
| Маржа | ~90–95% |
| Для $3k MRR | ~45 клиентов на базовом |
| Для $5k MRR | ~75 клиентов на базовом или ~50 на миксе |

**Риск:** API-интеграции ломаются, Google меняет авторизацию. Поддержка интеграций — постоянная нагрузка. MVP может быть сложнее, чем кажется, из-за OAuth-потоков.

### Идея 4: Микро-CRM для фрилансеров с трекингом оплат

**Целевая аудитория:** Фрилансеры и соло-консультанты (дизайнеры, разработчики, копирайтеры, маркетологи) с 5–30 постоянными клиентами.

**Проблема:** Фрилансеры ведут клиентов в Google Sheets, Notion или в голове. Забывают выставить счёт, не отслеживают просроченные оплаты, не видят, сколько заработали за квартал. CRM типа HubSpot или Pipedrive — избыточны и дороги для одного человека.

**Почему заплатят быстро:** Деньги, которые забыли попросить — самая болезненная проблема фрилансера. Если инструмент напоминает "Клиент X не платил 14 дней, вот шаблон напоминания" — это прямая ценность.

**Минимальный MVP:**
- Список клиентов с контактами и заметками
- Список проектов, привязанных к клиентам
- Трекинг статусов оплат: выставлен → оплачен → просрочен
- Автоматические напоминания об оплате (email клиенту)
- Дашборд: MRR, выручка за месяц/квартал, должники

**Стек технологий:**
- Next.js (full-stack)
- SQLite через Turso (бесплатный tier, проще PostgreSQL для простых данных)
- Clerk или Supabase Auth (авторизация)
- Resend (email)
- Stripe (биллинг самого SaaS)

**Как получить первых 20 клиентов:**
- Telegram-чаты фрилансеров (их сотни, особенно в рунете)
- Twitter/X — посты для англоязычных фрилансеров
- Product Hunt
- Пост на Habr/vc.ru: "Я потерял $X из-за забытых счетов — и написал инструмент"

**Финансовая модель:**
| Параметр | Значение |
|---|---|
| Цена | $12/мес или $99/год |
| Себестоимость | ~$0.5–1/мес |
| Маржа | ~92–95% |
| Для $3k MRR | ~250 клиентов |
| Для $5k MRR | ~420 клиентов |

**Риск:** Низкий ARPU. Нужно много клиентов для значимой выручки. Фрилансеры — ценочувствительная аудитория, склонная к бесплатным альтернативам.

### Идея 5: Сервис отзывов и социальных доказательств (Testimonial Collector)

**Целевая аудитория:** Любой малый бизнес, продающий услуги: агентства, консультанты, онлайн-школы, SaaS на ранней стадии.

**Проблема:** Собрать отзывы клиентов — мучение. Нужно попросить, дождаться, отформатировать, разместить на сайте. Большинство бизнесов имеют довольных клиентов, но нулевые социальные доказательства на сайте.

**Почему заплатят быстро:** Отзывы напрямую влияют на конверсию лендингов. Каждый маркетолог это знает. Инструмент, который делает сбор отзывов автоматическим, продаёт себя через демонстрацию: "Вот форма — отправьте клиенту — получите отзыв с фото — вставьте виджет на сайт".

**Минимальный MVP:**
- Создание формы для сбора отзывов (текст + фото/видео + рейтинг)
- Уникальная ссылка на форму для отправки клиентам
- Модерация: одобрить/отклонить отзыв
- Embeddable виджет (Wall of Love) для вставки на сайт — JS-сниппет
- Страница-витрина всех отзывов (публичная ссылка)

**Стек технологий:**
- Next.js
- Supabase (БД + storage для фото)
- Cloudflare R2 или Supabase Storage (хранение медиа)
- Vanilla JS виджет (embed-скрипт)
- Vercel

**Как получить первых 20 клиентов:**
- Запустить бесплатный план (до 5 отзывов), конвертировать в платный
- Twitter/X: показать пример виджета — визуальный контент вирусится
- Product Hunt (идеальная категория)
- Холодные DM агентствам и SaaS-фаундерам: "У вас нет отзывов на сайте — вот инструмент за 2 минуты"
- Партнёрская программа: 30% комиссия, привлечь 5–10 микроблогеров

**Финансовая модель:**
| Параметр | Значение |
|---|---|
| Цена | $19/мес (до 30 отзывов, 1 виджет), $39/мес (безлимит, видео, кастомизация) |
| Себестоимость | ~$1–3/мес |
| Маржа | ~90–95% |
| Для $3k MRR | ~110 клиентов на базовом |
| Для $5k MRR | ~185 клиентов на базовом |

## 3. Финальный выбор: Идея 1 — Генератор и трекер коммерческих предложений

### Почему именно эта идея

**По критерию скорости (30–60 дней до первых оплат):**
- MVP реально собрать за 2–3 недели: редактор на базе Tiptap/BlockNote, генерация ссылки, пиксель-трекинг через 1x1 img или beacon API, email-уведомления
- Никаких сложных интеграций с внешними API (в отличие от Идеи 3)
- Не нужна инфраструктура для фоновых задач на старте (в отличие от Идеи 2)
- Моментально понятная ценность: "Узнай, когда клиент открыл твоё КП"

**По критерию простоты:**
- Монолит на Next.js, одна база, один деплой на Vercel
- Нет сложной бизнес-логики, нет real-time, нет тяжёлых вычислений
- Трекинг открытий — технически тривиален (tracking pixel + webhook)
- Авторизация через Supabase Auth или Clerk — коробочное решение

**По критерию вероятности первых оплат:**
- Целевая аудитория (агентства, B2B-продажники) привыкла платить за инструменты продаж
- Прямая связь с деньгами: "Этот инструмент помогает закрывать сделки"
- Средний чек $29–59/мес — приемлем для агентства, но даёт хороший ARPU
- Конкуренты (PandaDoc, Qwilr, Better Proposals) стоят $35–65/мес за пользователя и перегружены функциями. Позиционирование: "Простая альтернатива за полцены"

### Реалистичный план на 60 дней

| Неделя | Действия |
|---|---|
| 1–2 | MVP: редактор, шаблоны (3 шт.), генерация ссылки, трекинг открытий |
| 3 | Биллинг (Stripe/Lemon Squeezy), лендинг, email-уведомления |
| 4 | Запуск: Product Hunt, 3–5 постов в Telegram/LinkedIn, 50 холодных DM агентствам |
| 5–6 | Итерация по фидбеку первых пользователей, добавление 2–3 шаблонов |
| 7–8 | Масштабирование аутрича: 20 DM в день, контент в соцсетях, партнёрства |

### Распределение бюджета ($1000)

| Статья | Сумма |
|---|---|
| Домен + email | $30 |
| Vercel Pro (если нужен) | $20/мес |
| Supabase (бесплатный tier → Pro при росте) | $0–25/мес |
| Stripe/Lemon Squeezy комиссии | ~3–5% от оборота |
| Дизайн (Figma, шрифты, иконки) | $0 (бесплатные ресурсы) |
| Маркетинг (мелкие расходы) | $50–100 |
| Резерв на 6 месяцев инфраструктуры | ~$300 |
| Итого первоначальных затрат | ~$200–400 |
| Остаток (runway) | ~$600–800 |

### Честная оценка рисков

1. **Главный риск — дистрибуция, не продукт.** Собрать MVP — не проблема. Проблема — найти 70 платящих клиентов. Это требует системного, ежедневного аутрича.
2. **Конкуренция.** PandaDoc — монстр с $100M+ ARR. Но малый бизнес не хочет платить $49/мес/пользователь за 90% функций, которые не использует. Ниша "простой и дешёвый" — реальна.
3. **Churn.** Если клиент не отправляет КП регулярно, он отпишется. Нужна аудитория с постоянным потоком предложений (агентства, аутсорс).
4. **Рост.** После первых 50–100 клиентов органический рост замедлится. Нужен контент-маркетинг или SEO — это длинная игра.

### Итог

Это не "идея на миллиард". Это прагматичный микро-SaaS, который один человек может запустить за месяц, начать получать оплаты на второй месяц, и довести до $3–5k MRR за 4–6 месяцев при системной работе над дистрибуцией. Если не получится — потеряете $200–400 и 2 месяца времени. Это приемлемая цена эксперимента.
