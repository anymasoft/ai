## 4. КЛАССИФИКАЦИЯ ОПЕРАЦИЙ

### УРОВЕНЬ 1: БЕЗОПАСНЫЕ (Green)
```
Условия:
- Обратимые операции
- Не требуют 2FA
- Не требуют логирования
- Не требуют одобрения

Примеры:
✅ Theme
✅ Language
✅ enterToSend
✅ autoScroll
✅ showThinking
✅ speechToText
✅ cloudBrowserVoices (с warning)

Действие: Прямое сохранение в localStorage
```

### УРОВЕНЬ 2: ВАЖНЫЕ (Yellow)
```
Условия:
- Частично обратимые операции
- Требуют одобрения/подтверждения
- Требуют логирования
- Требуют валидации

Примеры:
⚠️ Avatar загрузка (нужна валидация)
⚠️ TwoFactorAuthentication (нужна верификация)
⚠️ ImportConversations (нужна валидация JSON)
⚠️ SharedLinks управление

Действие: Подтверждение диалогом → API сохранение → Логирование
```

### УРОВЕНЬ 3: КРИТИЧНЫЕ (Red)
```
Условия:
- Необратимые операции
- Требуют 2FA верификации
- Требуют явного согласия
- Требуют обязательного логирования
- Могут разрушить приложение/данные

Примеры:
🔴 DeleteAccount (аккаунт + все данные)
🔴 ClearChats (все разговоры)
🔴 RevokeKeys (все API ключи)
🔴 AgentApiKeys (доступ к ключам)
🔴 DeleteAccount (для локальных пользователей)

Действие: Диалог → 2FA верификация → Email подтверждение → Двойное подтверждение → API операция → Логирование + Алерт
```

## 6. КРИТИЧНЫЕ НАЙДЕННЫЕ ПРОБЛЕМЫ

### Проблема 1: DeleteAccount без 2FA
```
РИСК: 🔴 КРИТИЧНЫЙ
ВЕРОЯТНОСТЬ: Высокая (если скомпрометирован email)
ВЛИЯНИЕ: Удаление аккаунта и всех данных

СЦЕНАРИЙ АТАКИ:
1.攻击者 получает доступ к браузеру пользователя
2. Открывает Settings → Account → Delete Account
3. Email подтверждение уже отправлено (пользователь в браузере)
4. Диалог закрывается
5. Аккаунт удалён БЕЗ уведомления пользователя

РЕШЕНИЕ:
1. Добавить 2FA верификацию
2. Отправить SMS уведомление
3. Добавить grace period (24ч)
4. Требовать password confirm
```

### Проблема 2: ClearChats без 2FA
```
РИСК: 🔴 КРИТИЧНЫЙ
ВЕРОЯТНОСТЬ: Средняя (требует открыть диалог)
ВЛИЯНИЕ: Потеря всех разговоров

СЦЕНАРИЙ АТАКИ:
1. Неавторизованный доступ к браузеру
2. Settings → Data → Clear All Chats
3. Нажать "Delete" дважды
4. Все разговоры удалены

РЕШЕНИЕ:
1. Добавить 2FA верификацию
2. Создавать автоматический бэкап перед удалением
3. Держать в "корзине" 30 дней
4. Требовать email подтверждение
```

### Проблема 3: AgentApiKeys видны в UI
```
РИСК: 🔴 КРИТИЧНЫЙ
ВЕРОЯТНОСТЬ: Очень высокая (screenshot, DevTools)
ВЛИЯНИЕ: Компрометация всех интеграций

СЦЕНАРИЙ АТАКИ:
1. Пользователь делает screenshot настроек
2. Отправляет по email/Slack
3. Ключи становятся видны в истории
4. Злоумышленник использует ключи

РЕШЕНИЕ:
1. Скрывать полные ключи (показывать ***** или последние 4 символа)
2. Требовать 2FA для просмотра полного ключа
3. Копировать в буфер одним кликом
4. Логировать все просмотры
5. Ограничить IP адреса для использования ключей
```

### Проблема 4: ImportConversations без валидации
```
РИСК: 🔴 КРИТИЧНЫЙ
ВЕРОЯТНОСТЬ: Средняя (требует загрузить JSON)
ВЛИЯНИЕ: XSS, инъекция кода, потеря данных

СЦЕНАРИЙ АТАКИ:
1. Пользователь загружает JSON с XSS payload
2. JSON парсится без санитизации
3. Payload выполняется при отображении разговора
4. Компрометация сессии

РЕШЕНИЕ:
1. Валидировать JSON структуру
2. Санитизировать все текстовые поля
3. Проверять максимальный размер (50MB)
4. Показывать превью перед импортом
5. Требовать администраторское одобрение
6. Логировать все импорты
```

### Проблема 5: Balance параметры для пользователя
```
РИСК: 🟠 СРЕДНИЙ
ВЕРОЯТНОСТЬ: Низкая (read-only)
ВЛИЯНИЕ: Путаница, неправильное использование

СЦЕНАРИЙ:
1. Пользователь видит autoRefillEnabled = true
2. Не понимает, как изменить параметр
3. Контактирует поддержку
4. Путаница с правами доступа

РЕШЕНИЕ:
1. Перенести auto-refill параметры в админку
2. Показать пользователю только текущий баланс
3. Добавить ссылку на админку для изменения
```
