# Запуск веб-интерфейса Crawl4AI локально (без Docker)

Если вы склонировали репозиторий и хотите запустить веб-интерфейс (дашборд мониторинга и playground) на своём компьютере, выполните следующие шаги.

## Предварительные требования

- Python 3.10 или выше
- Установленный `pip`
- Браузер Chromium (установится автоматически через Playwright)

## Шаг 1: Установка зависимостей

Перейдите в корневую папку проекта (где находится `setup.py`).

```bash
cd /путь/к/crawl4ai
```

Установите пакет `crawl4ai` в режиме разработки (если ещё не установлен):

```bash
pip install -e .
```

Установите дополнительные зависимости для сервера:

```bash
pip install -r deploy/docker/requirements.txt
```

Установите браузер Playwright:

```bash
python -m playwright install --with-deps chromium
```

## Шаг 2: Настройка окружения (опционально)

Если вы планируете использовать LLM (например, для вопросов или экстракции), создайте файл `.llm.env` в корне проекта (скопируйте пример):

```bash
cp deploy/docker/.llm.env.example .llm.env
```

Отредактируйте `.llm.env`, добавив свои API-ключи (OpenAI, Anthropic и т.д.). Если не нужны LLM, можно пропустить.

## Шаг 3: Запуск сервера

Перейдите в папку `deploy/docker`:

```bash
cd deploy/docker
```

Запустите сервер FastAPI:

```bash
python server.py
```

Сервер запустится на `http://0.0.0.0:8020` (согласно конфигурации `config.yml`). Однако в конфиге указан порт 8020, но в Docker используется порт 11235. Проверьте, какой порт используется в выводе консоли.

Обычно сервер печатает сообщение:

```
MCP server running on 0.0.0.0:8020
```

Значит, веб-интерфейс будет доступен по адресу `http://localhost:8020`.

## Шаг 4: Откройте веб-интерфейс

В браузере перейдите по одному из адресов:

- **Playground (интерактивная площадка)**: `http://localhost:8020/playground`
- **Дашборд мониторинга**: `http://localhost:8020/dashboard`
- **Корневой URL** (`http://localhost:8020`) перенаправит на playground.

Если порт отличается (например, 11235), используйте его.

## Шаг 5: Использование

- В **Playground** можно вводить URL, настраивать параметры краулинга и сразу получать результат.
- В **Дашборде** можно наблюдать за системными метриками, активными запросами и состоянием браузерного пула.
- Также доступны REST API endpoints (документация Swagger UI: `http://localhost:8020/docs`).

## Альтернативный запуск через uvicorn

Можно запустить сервер напрямую через uvicorn (так делает `server.py`):

```bash
uvicorn server:app --host 0.0.0.0 --port 11235 --reload
```

Это запустит сервер на порту 11235 (как в Docker). Убедитесь, что вы находитесь в папке `deploy/docker`.

## Возможные проблемы

1. **Ошибка импорта модулей**: Убедитесь, что вы установили `crawl4ai` в режиме разработки (`pip install -e .`) из корня проекта.

2. **Ошибка Playwright**: Если браузер не установлен, выполните `python -m playwright install chromium`.

3. **Порт занят**: Если порт 8020 или 11235 занят, измените порт в `config.yml` (параметр `app.port`) или передайте аргумент `--port` в uvicorn.

4. **Отсутствует Redis**: Сервер использует Redis для кэширования и мониторинга. Если Redis не установлен, могут возникнуть ошибки. Установите Redis локально или запустите его в Docker:

   ```bash
   docker run -d -p 6379:6379 redis:alpine
   ```

   Либо измените конфигурацию Redis в `config.yml` на `memory://` (временное хранилище).

## Краткая справка по командам

```bash
# Установка
cd /path/to/crawl4ai
pip install -e .
pip install -r deploy/docker/requirements.txt
python -m playwright install chromium

# Запуск сервера
cd deploy/docker
python server.py

# Или с uvicorn
uvicorn server:app --host 0.0.0.0 --port 11235 --reload
```

После запуска откройте браузер и перейдите по адресу `http://localhost:8020` (или другому порту, который указан в выводе).

Теперь вы можете пользоваться веб-интерфейсом Crawl4AI без Docker!
