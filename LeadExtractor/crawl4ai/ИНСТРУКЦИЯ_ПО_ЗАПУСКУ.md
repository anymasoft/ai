# Инструкция по запуску и использованию Crawl4AI

Crawl4AI — это открытый веб-краулер и скрейпер, предназначенный для преобразования веб-страниц в чистый Markdown, готовый для использования в RAG, агентах и конвейерах данных. Проект предоставляет несколько способов запуска: локально через Python, через CLI и через Docker с веб-интерфейсом.

## Быстрый старт

Если вы хотите быстро попробовать Crawl4AI, выполните следующие шаги:

1. Установите пакет:
   ```bash
   pip install -U crawl4ai
   ```

2. Запустите пост-установочную настройку:
   ```bash
   crawl4ai-setup
   ```

3. Проверьте установку:
   ```bash
   crawl4ai-doctor
   ```

4. Запустите простой краулинг через CLI:
   ```bash
   crwl https://www.nbcnews.com/business -o markdown
   ```

Или используйте Docker для запуска с веб-интерфейсом:
   ```bash
   docker pull unclecode/crawl4ai:latest
   docker run -d -p 11235:11235 --name crawl4ai --shm-size=1g unclecode/crawl4ai:latest
   ```
   Затем откройте в браузере `http://localhost:11235/dashboard`.

## Установка

### Установка через pip

**Базовая установка** (рекомендуется для большинства пользователей):
```bash
pip install crawl4ai
crawl4ai-setup
```

**Установка с дополнительными функциями** (например, PyTorch, трансформеры):
```bash
pip install crawl4ai[torch]          # с поддержкой PyTorch
pip install crawl4ai[transformer]    # с трансформерами
pip install crawl4ai[cosine]         # с косинусным сходством
pip install crawl4ai[all]            # все дополнительные функции
```

**Установка для разработчиков** (из исходного кода):
```bash
git clone https://github.com/unclecode/crawl4ai.git
cd crawl4ai
pip install -e .
```

**Примечание:** После установки может потребоваться установить браузерные зависимости Playwright:
```bash
python -m playwright install --with-deps chromium
```

### Установка через Docker

Crawl4AI предоставляет готовый Docker-образ с веб-интерфейсом, мониторингом и API.

1. Получите последний образ:
   ```bash
   docker pull unclecode/crawl4ai:latest
   ```

2. Запустите контейнер:
   ```bash
   docker run -d -p 11235:11235 --name crawl4ai --shm-size=1g unclecode/crawl4ai:latest
   ```

   Параметры:
   - `-p 11235:11235` — проброс порта (веб-интерфейс будет доступен на localhost:11235)
   - `--shm-size=1g` — увеличение разделяемой памяти для браузера
   - `--name crawl4ai` — имя контейнера

3. Проверьте, что контейнер запущен:
   ```bash
   docker ps
   ```

4. Откройте веб-интерфейс в браузере:
   - Дашборд мониторинга: `http://localhost:11235/dashboard`
   - Интерактивная площадка: `http://localhost:11235/playground`
   - Корневой URL перенаправляет на площадку.

## Запуск проекта

### Локальный запуск (Python API)

Используйте асинхронный API для краулинга в своих Python-скриптах.

Пример `simple_crawl.py`:
```python
import asyncio
from crawl4ai import AsyncWebCrawler

async def main():
    async with AsyncWebCrawler() as crawler:
        result = await crawler.arun(
            url="https://www.nbcnews.com/business",
        )
        print(result.markdown)

if __name__ == "__main__":
    asyncio.run(main())
```

Запустите скрипт:
```bash
python simple_crawl.py
```

### Запуск через CLI

После установки доступна команда `crwl` (или `crawl4ai`).

**Основные команды:**
- `crwl <URL>` — простой краулинг страницы
- `crwl <URL> -o markdown` — вывод в формате Markdown
- `crwl <URL> -o json` — вывод в формате JSON
- `crwl <URL> -q "Ваш вопрос"` — задать вопрос по содержимому страницы (использует LLM)
- `crwl <URL> -j` — извлечение структурированных данных через LLM
- `crwl profiles` — управление профилями браузера
- `crwl cdp` — запуск браузера с отладкой через CDP
- `crwl browser start` — запуск встроенного браузера
- `crwl examples` — показать примеры использования

**Примеры:**
```bash
# Базовый краулинг с выводом Markdown
crwl https://example.com -o markdown

# Глубокий краулинг с BFS (максимум 10 страниц)
crwl https://docs.crawl4ai.com --deep-crawl bfs --max-pages 10

# Извлечение данных с помощью LLM
crwl https://www.example.com/products -j "Extract all product prices"

# Вопрос по содержимому
crwl https://example.com -q "What is the main topic?"
```

### Запуск через Docker (с веб-интерфейсом)

После запуска контейнера (см. выше) доступны:

1. **Дашборд мониторинга** (`http://localhost:11235/dashboard`) — показывает метрики системы, состояние браузерного пула, активные запросы, использование памяти и т.д.
2. **Интерактивная площадка** (`http://localhost:11235/playground`) — позволяет тестировать краулинг через веб-интерфейс, настраивать параметры и сразу видеть результат.
3. **REST API** — доступен на том же порту. Документация API доступна через Swagger UI по адресу `http://localhost:11235/docs`.

**Пример использования API:**
```python
import requests

response = requests.post(
    "http://localhost:11235/crawl",
    json={"urls": ["https://example.com"], "priority": 10}
)
print(response.json())
```

## Веб-интерфейс

### Дашборд мониторинга

Дашборд предоставляет реальную информацию о работе Crawl4AI:

- **Системное здоровье**: загрузка CPU, память, сеть, время работы.
- **Активные запросы**: список текущих краулингов.
- **Браузерный пул**: количество браузеров в постоянном, горячем и холодном пулах, использование памяти.
- **Статистика эндпоинтов**: количество запросов, ошибки, среднее время ответа.
- **Логи уборщика (janitor)**: события автоматической очистки ресурсов.
- **Логи ошибок**: последние ошибки с контекстом.

Доступ к дашборду: `http://localhost:11235/dashboard`

### Интерактивная площадка (Playground)

Площадка позволяет:

- Ввести URL и настроить параметры краулинга (стратегия, фильтры, извлечение).
- Запустить краулинг и сразу увидеть результат в виде Markdown, HTML или JSON.
- Тестировать LLM-экстракцию с предустановленными схемами.
- Генерировать код для Python, cURL, Node.js на основе настроек.

Доступ к площадке: `http://localhost:11235/playground`

## Основные команды CLI

Полный список команд можно получить через `crwl --help`.

| Команда | Описание |
|---------|----------|
| `crwl <URL>` | Краулинг страницы с настройками по умолчанию |
| `crwl crawl <URL>` | То же, но с явным указанием команды crawl |
| `crwl crawl <URL> -o json` | Вывод в JSON |
| `crwl crawl <URL> -o markdown-fit` | Вывод отфильтрованного Markdown |
| `crwl crawl <URL> --deep-crawl bfs --max-pages 20` | Глубокий краулинг с BFS |
| `crwl crawl <URL> -j` | LLM-экстракция (запросит провайдера при первом использовании) |
| `crwl crawl <URL> -q "Вопрос"` | Вопрос-ответ по содержимому |
| `crwl profiles` | Интерактивный менеджер профилей браузера |
| `crwl cdp` | Запуск браузера с CDP-отладкой |
| `crwl browser start` | Запуск встроенного браузера |
| `crwl browser status` | Статус встроенного браузера |
| `crwl browser stop` | Остановка встроенного браузера |
| `crwl config list` | Показать глобальную конфигурацию |
| `crwl config set VERBOSE true` | Установить параметр конфигурации |
| `crwl examples` | Показать примеры использования |

## Примеры использования

### Простой краулинг

```bash
crwl https://news.ycombinator.com -o markdown > hackernews.md
```

### Глубокий краулинг

Используйте стратегии BFS, DFS или Best-First для обхода нескольких страниц.

```bash
crwl https://docs.crawl4ai.com --deep-crawl bfs --max-pages 30
```

### Извлечение структурированных данных с помощью LLM

Перед первым использованием LLM нужно настроить провайдера (например, OpenAI, Anthropic, Ollama). Команда запросит API-токен и сохранит его в `~/.crawl4ai/global.yml`.

```bash
crwl https://openai.com/api/pricing/ -j "Extract model names and pricing"
```

Можно использовать локальную модель Ollama:

```bash
crwl config set DEFAULT_LLM_PROVIDER "ollama/llama3.3"
crwl config set DEFAULT_LLM_PROVIDER_TOKEN "no-token"
crwl https://example.com -j "Extract data"
```

## Управление профилями браузера

Crawl4AI позволяет создавать персистентные профили браузера для аутентифицированного краулинга (например, для сайтов, требующих логина).

1. Запустите менеджер профилей:
   ```bash
   crwl profiles
   ```

2. В интерактивном меню выберите "Create new profile". Откроется браузер, где можно выполнить вход на нужный сайт, после чего нажмите `q` в терминале для сохранения профиля.

3. Используйте профиль для краулинга:
   ```bash
   crwl https://site-requiring-login.com/dashboard -p my-profile-name
   ```

## Настройка конфигурации

Глобальная конфигурация хранится в `~/.crawl4ai/global.yml`. Её можно редактировать вручную или через CLI:

```bash
crwl config list                          # показать все настройки
crwl config get DEFAULT_LLM_PROVIDER      # получить значение
crwl config set BROWSER_HEADLESS false    # установить значение
```

Основные настройки:
- `DEFAULT_LLM_PROVIDER` — провайдер LLM (например, "openai/gpt-4o")
- `DEFAULT_LLM_PROVIDER_TOKEN` — API-токен
- `BROWSER_HEADLESS` — запускать браузер в headless-режиме (true/false)
- `USER_AGENT_MODE` — режим User-Agent ("random", "desktop", "mobile")
- `VERBOSE` — подробный вывод (true/false)

## Решение проблем

### Ошибки браузера

Если возникают ошибки, связанные с Playwright, переустановите браузер:

```bash
python -m playwright install chromium
```

Или запустите `crawl4ai-setup` повторно.

### Проблемы с памятью

При краулинге многих страниц или использовании Docker может потребоваться увеличить лимиты памяти.

Для Docker:
```bash
docker run -d -p 11235:11235 --name crawl4ai --shm-size=2g -m 4g unclecode/crawl4ai:latest
```

Для локального запуска можно ограничить количество одновременных страниц через настройку `MAX_PAGES` в конфигурации.

### Ошибки SSL

Если сайты используют самоподписанные сертификаты, добавьте в конфигурацию браузера `ignore_https_errors=True`.

### Логи

Логи по умолчанию пишутся в stderr. Для более детального логирования установите `VERBOSE=true` в конфигурации или используйте флаг `-v` в CLI.
